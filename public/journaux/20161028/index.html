Ce matin (ou ce qu’il en reste)

On va essayer un kata bowling JavaScript en structurant le code en respectant le paradigme fonctionnel.

  • Avec [[Jasmine]] `describe(“élément décrit”, function () {} );“.
  • Condition sur les type en [[JavaScript]] : var a = ["e"]; typeof(a);.
  • On écrit apparement le même nombres de tests en fonctionnel qu’en objet (pour le moment).
  • Une fonction est une action de transformation, la base de la programmation : transformer des données
  • On parle de l’approche Top-Down, en fonctionnel
  • On compare le nombre de mot clefs entre [[JavaScript]] et [[Haskell]] sur un exemple simple.
  • [[Hafid]] a tenter de faire (truc) -> {} mais ça ne fonctionne pas dans Firefox.
